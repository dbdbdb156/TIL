# Call by Value vs Call by Reference
### 1. Call by value
: 함수 호출시 전달되는 변수 값을 복사해서 함수 인자로 전달함
즉, 함수 안에서 인자값이 변경 되더라도 외부 변수 값은 변경 안됨
- 인자를 넘겨줄때마다 메모리 공간을 할당해야되서 메모리 공간 추가

### 2. Call by reference
: 함수 호출시 인자로 전달되는 변수의 레퍼런스를 전달함
즉, 함수 안에서 인자 값이 변경되면, Argument 로 전달된 객체의 값도 변경됨
- 원본값이 변할수 있음

* java의 경우 항상 call by value로 값을 넘김
; c/c++ 과 같이 변수의 주소값 자체를 가져올 방법이 없음
- reference type을 넘길 시에는 해당 객체의 주소값을 복사하여 이를 가지고 사용한다.
(주소값도 복사해서 사용하기에 call by value. 원본 객체 자체를 변경할 수는 없다)

** 쉽게 말하면 java 는 데이터를 저장하는 위치가 따로 있고, 그 데이터의 위치 주소값을 변수에 할당하는것이기 때문에, 변수의 값이 바뀐다는 것은 데이터의 값이 바뀌는 것이 아닌 데이터의 위치 주소값이 변하는 것이다.
