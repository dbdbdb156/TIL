# 알고리즘
## 동적계획법
### 종만북

#### 1. DP 란
정의 : 어떤 부분 문제를 풀때, 여러번 계산하는 대신 한번만 계산하고 계산 결과를 재활용하여 속도 향상목적
- 메모라이제이션이 필수

* 구현패턴
1) 문제를 재귀적으로 해결하는 완전 탐색 알고리즘 만들기
2) 중복된 부분을 한번만 계산하도록 메모라이제이션 적용

#### 2. 최적화 문제들
정의 : 여러 개의 가능한 답중 가장 좋은 답(최적해)를 찾아내는 문제
* 유의점
1) 파라미터 설계시, 해결이 되어야 하는 입력들과 vs 결정이 난 입력들로 나누기.
: 결정된 입력들은 사실상 없어도 됨.( 결과값은 return 으로 넘겨줄수 있음)

##### 최적 부분 구조
: 각 부분 문제의 최적해만 있으면 전체 문제의 최적해를 쉽게 얻어낼 수 있을 경우

* 구현 패턴
1) 완전 탐색 알고리즘 설계
2) 전체 답의 점수를 반환 하는것이 아닌, 앞으로 남은 선택들에 해당 하는 점수만을 반환하도록 부분 문제 정의
3) 재귀 호출의 입력 이전의 선택의 관련된 정보가 있다면 꼭 필요한 것만 남기고 줄이기.
-> 즉, 가능한 한 중복되는 부분 문제를 많이 만드는것이 목적
4) 입력이 배열이거나 문자열인 경우, 적절한 변환을 통해 메모라이제이션 할수 있도록 하기
5) 메모라이제이션 적용

* 유의 사항 : 오버플로우를 유의 해야한다.
즉, 값들의 합,곱,뺄셈을 통해서 32비트의 범위(20억 쯤,2^31-1)를 넘게 된다면 Long 형 또는 DOUBLE 형으로 선언해주어야 함.
