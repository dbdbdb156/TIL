# 알고리즘
## 그리디
### 백준

1. 19598 최소 회의실 개수
- 풀이 : 우선순위 큐를 이용해서, end 값들만 관리. end 값의 리스트가 들어오는 start 값보다 큰 경우만 큐에 유지해주며 크기의 최대값구하기
- 근거 : end 값들을 올림차순으로 정렬하여 관리할수 있는 자료구조가 필요함. 그래서 우선순위큐를 사용함

https://www.acmicpc.net/problem/19598

2. 21758 꿀 따기
- 풀이 : 그리디를 이용해서, 3가지 경우로 나눠서 최대값 구하기. presum 을 통해서 부분합 구하기
1) 벌통이 중간, 양 끝에 벌
2) 벌통이 왼쪽끝 , 벌들은 오른쪽 끝, 중간에 하나
3) 벌통이 오른쪽 끝, 벌들은 왼쪽 끝, 중간에 하나
- 근거 :
1) 최대수가 10만이기에 O(n^2) 으로 풀면 시간초과가 나옴
: sweeping 을 통해서 읽으면 O(n)으로 해결 가능
2) 가장 긴 길이의 합을 구하기 위해서, 벌통과 벌 한마리는 제일 끝쪽에 나누는 경우를 생각함.
: 만약, 벌이 맨 끝에 있지 않다면, 어차피 더하지 않기때문에 더 작은 값이 나오므로 최대값을 구할수 없음.
3) presum 배열을 유지해준 이유는 처음 O(n) 으로 생성해두면 참조는 O(1)로서 구간 합을 구할수 있음.

https://www.acmicpc.net/problem/21758

3. 20365 블로그2
- 풀이 : 그리디를 이용해서 B,R 의 숫자가 더 작은 값에 +1 을 한 값이 결과
- 근거 : 
1) R이 많을지, B 가 많을지 확인할수 없음. 확실한 것은 blue 에서 red 로 바뀌는 때와, red 에서 blue 로 바뀌는 차이는
k : k+1 or k+1 : k 임.
2) n의 최대값이 50만이기에 재귀를 이용해서 blue 에서 red 로 바뀌고, red에서 blue로 바뀌는 것을 구현했을때 O(n^2)으로 시간초과가 남
: 즉 O(n) or, O(nlogn) 의 알고리즘이 필요함.

https://www.acmicpc.net/problem/20365

4. 16953 A -> B
- 풀이 : 재귀함수를 이용하여 파라미터로는 상태를 관리하는 value 의 값과 depth 로서 최소값 구하기
- 근거
1) A,B 의 값이 10억임. 하지만, 문제에서 곱하기 2와 1을 오른쪽에 추가하는것은 곱하기 10이기에 O(logn) 의 시간 복잡도를 가짐.
그렇기에 재귀를 통해서 문제를 품.
2) 값 변수를 Long 형으로 선언한 이유는 최대값인 10억일때, 1을 오른쪽에추가하는 연산을 하면 수가 100억이 됨.
 즉, int 값의 범위인 2^31-1( 대략 20억)을 넘기에 Long형으로 선언

https://www.acmicpc.net/problem/16953

5. 11047 동전 0
- 풀이 : 그리디를 통해서 k값 자체로서 원하는 값을 동전의 내림차순 순서대로, 즉 큰 순서대로 동전을 선택하며,
k % 동전 값 과 k/ 동전 값 연산을 적절히 취함.
- 근거 :
1) A1 = 1 이고 A2 이상의 동전은 이전 배열의 배수임. 즉, 작은 동전을 여러개 선택이 가능하면, 작은 수의 동전이 선택 가능함
2) 통상적인 동전 문제와 같이 DP 를 사용하며 확인하지 않은 이유 : 동전값의 최대값이 1억임.
: 배열을 1억개를 선언할수 없을뿐더라. A1=1 이기에 연산을 1억번 한다는 것도 과부화가 큼
3) A1 = 1 이기에 동전의 수가 선택되지 않는경우는 반드시 없음. 즉, 해가 무조건 존재함.

https://www.acmicpc.net/problem/11047
