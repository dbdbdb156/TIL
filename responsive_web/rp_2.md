# 미디어쿼리 와 뷰포트
## 1. 미디어쿼리
: 화면의 크기나 환경에 따라 웹사이트를 변경하는 기술
- 미디어 쿼리는 미디어별로 적용할 CSS 를 따로 작성, 미디어 유형이 필요함
ex) 
1) 링크방식 1 : css 파일 별도 관리, 속도 빠름.
<link rel="stylesheet" href="mediaqueries.css"
: CSS 파일 별도 관리시 브라우저의 기억장치 또는 캐싱의 특성을 이용해
웹사이트 재사용시 웹속도가 빨라짐.

2) 링크방식 2
<link rel="stylesheet" media="all and (min-width:320px)" href="sytle320px.css">
-> 안하는 편이 좋음.(css 파일 갯수가 늘어나서 웹사이트 속도가 느려짐)
3) 문서 내에 작성하는 방식
: html 문서내에 작성하기에 문서 용량 커짐. 웹 느려짐

** 주의사항
- 띄어쓰기 주의
- min/max 사용 : min은 작은것부터, max 는 큰것부터
- 브라우저 크기 감지 : html 문서의 화면 크기를 기준으로 감지


기초문법
@media[only 또는 not][미디어 유형][and 또는 , 콤마] (조건문){실행문} 
; 조건문에 따라 다른 화면이 나올수 있다. CSS 파일을 만들어 Link 로 참조하는편이 좋음.


*** style 의 값은 상속의 개념과 같이 부모에서 적용해두면 자녀에 모두 똑같이 적용된다.
즉, 부모에 적용해두고 세부적인 부분만 자식의 style 을 제어하면되는것이다.


## 2. 뷰포트
: 화면의 보이는 영역을 다루는 기술, 뷰포트
-실제 내용이 표시되는 영역, 데스크톱은 해상도, 스마트기기는 기본으로 설정되있는 값 

#### 뷰포트문법) ** 잘 외워둬야함

- width : device-width,양수
- height : device-height,양수
- initial-scale : 양수 / 뷰포트의 초기 배율 기본값 1
- user-scalable : yes or no / 뷰포트의 확대,축소 여부 지정, yes 하면 페이지 확대 가능
- minimum-scale : 양수 / 기본값 0.25
- maximum-scale : 양수 / 기본값 5.0

1. 데스크톱 뷰포트 영역 : 가변
2. 스마트기기 : 익스플로러(1024px)를 제외한 웹키트 기반 브라우저는 980px
* 메타 태그를 제외한 표준 방식으로 <style> 태그안에 선언 가능
